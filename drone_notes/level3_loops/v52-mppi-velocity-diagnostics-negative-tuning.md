# v52 MPPI 速度诊断和一轮反向排除

这轮还是只看 MPPI，不碰 PPO。

结果没有突破：

- smoke 101-105：`0%` 完赛，平均过门 `0.80`，`100%` crash；
- dev 1-10：`0%` 完赛，平均过门 `0.20`，`100%` crash。

这轮试了很多 MPPI 小改，但都没有保留：

- 门框 contact cost；
- 没对准时惩罚高速穿门；
- 更大 horizon / 更多 samples；
- 门前绕障 staging；
- 小幅抬高穿门目标；
- 更相信 guide 或更相信 MPPI；
- 只避让已观测障碍；
- 靠近门平面时限制门轴速度。

这些要么让 smoke mean gates 从 `0.80` 掉到 `0.40/0.20/0.60`，要么只是换了一批会过的 seed，没有整体变强。

保留下来的不是控制器改动，而是 evaluator 诊断改动：现在每个失败 episode
会记录终点速度，以及终点速度在目标门坐标系里的 `vx/vy/vz`。

新诊断说明：MPPI 不是只差一点位置。它撞门框时，横向和竖直速度也很大。
dev 失败时，平均 `|vy|` 和 `|vz|` 都在 `0.7 m/s` 左右。通俗说，就是飞机不是
慢慢贴到门框上，而是带着明显横向/竖直速度擦过去。

下一步应该做更结构化的 gate-frame velocity controller：在门坐标系里同时管
前进速度、横向速度、高度速度，而不是继续往当前 cost 上叠小惩罚。
