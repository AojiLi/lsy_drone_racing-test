# v52 MPPI 后续门预对齐和撞击分类修复

这轮还是只看 MPPI，不碰 PPO。

当前 MPPI 还没有接近通过 Level3：

- 101-105 smoke：`0%` 完赛，平均过门 `0.80`，`100%` contact；
- 1-10 dev：`0%` 完赛，平均过门 `0.20`，`100%` contact；
- 最好的单个 seed 还是 seed 103，能到第 2 个门附近，但没有完赛。

这轮真正有价值的是把失败原因看清楚了。之前 evaluator 容易把撞击后的
disabled 位置看成 `[-1, -1, -1]`，所以很多失败会被误标成
`bounds_or_ground`。现在改成用终止前一步的位置做分类，结果更清楚：

- smoke 里主要是 `near_gate_obstacle` 和 `gate_side_frame`；
- dev 里主要是 `gate_side_frame` 和 `gate_vertical_frame`。

通俗说：MPPI 不是简单飞出界，而是经常“冲门洞时擦门框”，或者“门前障碍物
和门洞选择没协调好”。

试过但回退的方向：

- 大幅绕障碍物；
- 门洞内偏置；
- 更强 obstacle clearance；
- 第一门预对齐；
- 起飞稳定分支；
- 更平滑动作滤波；
- 强制 near-plane 对齐。

这些都没有稳定提高 mean gates，有的还明显变差。

目前保留的只有一个小控制器改动：过了第一个门以后，如果下一门前横向或
高度误差还大，就先保守地对齐门前位置，速度放慢一点。

下一步不应该跑 101-200 hard eval，也不应该生成 PPO teacher data。应该继续
做 MPPI 本身，把它变成真正懂“门洞 + 障碍物 + 撞门框”的规划器：

1. 在门坐标系里控制穿门速度；
2. 在 MPPI cost 里提前惩罚侧框、上框、下框；
3. 避障必须和门洞可通过区域一起算，不能看见障碍就大幅绕；
4. 先用 trace 和小 seed 集合调到稳定多门，再扩大评估。
